# OLAP-система анализа эффективности использования библиотечного фонда

Репозиторий с материалами учебной практики НИР по теме:

**OLAP-система анализа эффективности использования библиотечного фонда**.

Работа выполняется по направлению **02.03.02 Фундаментальная информатика и информационные технологии**.

Проект показывает, как можно представить данные о библиотечном фонде в виде реляционной модели и получить первые показатели для OLAP-анализа.

В текущей версии подготовлены материалы для пятого и шестого этапов:

- демонстрационная реляционная база данных SQLite;
- измерения `Дата`, `Книга`, `Читатель`, `Филиал` и таблица фактов `Выдачи`;
- SQL-запросы для расчета показателей эффективности использования фонда;
- таблицы с метриками и графики для отчета.
- отчет по пятому этапу;
- черновик итогового отчета по шестому этапу.

Данные в базе являются демонстрационными. Они нужны для проверки структуры модели, запросов и набора показателей. На следующих этапах предполагается уточнить источники данных и расширить модель под реальные сведения библиотечной системы.

## Состав материалов

В каталоге `artifacts` находятся:

- `library_fund_olap.sqlite` - демонстрационная база данных;
- `table_branch_metrics.csv` - таблица показателей по филиалам;
- `table_category_metrics.csv` - таблица показателей по разделам фонда;
- `chart_loans_by_month.svg` - график динамики выдач по месяцам;
- `chart_turnover_by_category.svg` - график обращаемости фонда по разделам;
- `kt5_report.md` - текст отчета КТ5;
- `kt5_report.docx` - версия отчета для редактирования в Word;
- `kt5_report.html` - версия отчета с встроенными графиками.
- `kt6_draft_report.md` - текстовая версия черновика итогового отчета;
- `kt6_draft_report.docx` - черновик итогового отчета для шестого этапа.

В каталоге `sql` находится файл `analytical_queries.sql`. В нем приведены основные SQL-запросы, с помощью которых были получены таблицы и показатели для отчета.

## Как проверить базу

Базу `artifacts/library_fund_olap.sqlite` можно открыть в DB Browser for SQLite, DBeaver или другом редакторе SQLite. Для проверки достаточно посмотреть структуру таблиц и выполнить запросы из `sql/analytical_queries.sql`.
