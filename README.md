# OLAP-система анализа эффективности использования библиотечного фонда

Репозиторий с материалами учебной практики НИР по теме:

**OLAP-система анализа эффективности использования библиотечного фонда**.

Обучающийся: Брыляков Никита Евгеньевич.  
Направление подготовки: **02.03.02 Фундаментальная информатика и информационные технологии**.  
Трек: **П - прикладной, разработка программной системы**.

Проект показывает, как можно представить данные о библиотечном фонде в виде ROLAP-модели и получить аналитические показатели по филиалам, разделам фонда и времени.

## Что подготовлено

- демонстрационная база данных SQLite;
- SQL-запросы для расчета показателей;
- таблицы с метриками по филиалам и разделам фонда;
- графики для отчета;
- отчет по пятому этапу;
- черновик отчета по шестому этапу;
- итоговый отчет и презентация для седьмого этапа.

Данные в базе являются демонстрационными. Они нужны для проверки структуры модели, запросов и набора показателей. При дальнейшем развитии проект можно подключить к реальным выгрузкам автоматизированной библиотечной системы или электронного каталога.

## Состав материалов

В каталоге `artifacts` находятся:

- `library_fund_olap.sqlite` - демонстрационная база данных;
- `table_branch_metrics.csv` - показатели по филиалам;
- `table_category_metrics.csv` - показатели по разделам фонда;
- `chart_loans_by_month.svg` - график динамики выдач по месяцам;
- `chart_turnover_by_category.svg` - график обращаемости по разделам фонда;
- `kt5_report.docx` - отчет по пятому этапу;
- `kt6_draft_report.docx` - черновик итогового отчета по шестому этапу;
- `kt7_final_report.docx` - итоговый отчет по учебной практике;
- `kt7_presentation.pptx` - презентация к защите.

В каталоге `sql` находится файл `analytical_queries.sql` с основными SQL-запросами.

## Как проверить базу

Базу `artifacts/library_fund_olap.sqlite` можно открыть в DB Browser for SQLite, DBeaver или другом редакторе SQLite. Для проверки достаточно посмотреть структуру таблиц и выполнить запросы из `sql/analytical_queries.sql`.

Основная модель:

- `dim_date` - календарное измерение;
- `dim_branch` - измерение филиалов;
- `dim_book` - измерение фонда;
- `dim_reader` - измерение читателей;
- `fact_circulation` - таблица фактов выдачи.
